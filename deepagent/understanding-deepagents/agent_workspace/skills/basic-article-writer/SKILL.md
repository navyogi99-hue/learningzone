## Purpose

* Topic
* Target audience (optional)
* Desired article length (optional)
* Tone (optional)

## Output

A well-structured article in Markdown format.

---

## Instructions

### 1. Understand the Topic

* Read and understand the provided topic.
* Identify the main concept and key subtopics.
* If the topic is unclear, make reasonable assumptions.

### 2. Research the Topic

* Gather accurate and up-to-date information.
* Focus on:

  * Definition
  * Importance
  * Key concepts
  * Benefits
  * Challenges
  * Examples
  * Future trends (if applicable)

### 3. Create an Outline

Use the following structure:

```text
Title

Introduction

Main Section 1

Main Section 2

Main Section 3

Conclusion
```

### 4. Write the Article

#### Title

* Create a clear and descriptive title.

#### Introduction

* Introduce the topic.
* Explain why it matters.
* Keep it concise.

#### Body

* Divide content into logical sections.
* Use headings and subheadings.
* Explain concepts in simple language.
* Include examples where helpful.

#### Conclusion

* Summarize key points.
* End with a useful takeaway.

### 5. Formatting Rules

* Use Markdown headings.
* Use bullet points for lists.
* Keep paragraphs short.
* Avoid jargon when possible.
* Define technical terms when necessary.

### 6. Quality Checklist

Before finalizing:

* Article has a clear title.
* Introduction explains the topic.
* Content is factually accurate.
* Sections flow logically.
* Grammar and spelling are correct.
* Conclusion summarizes key ideas.
* Markdown formatting is valid.

## Example Usage

Input:

```yaml
topic: LangChain
audience: Beginners
length: 800 words
tone: Educational
```

Expected Output:

A beginner-friendly article explaining what LangChain is, why it is useful, its core components, common use cases, and a concluding summary.
